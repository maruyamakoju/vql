VQL

映像を一度コンパイルして、何度でも確定的に問い合わせる

YOLO・VLM 等の既存モデルを推論器ではなくインデクサとして使い、 映像から物体・軌跡・領域遷移・滞在・近接・順序関係を VIR（Video Intermediate Representation）へコンパイル。 その後の検索はモデル不要で<10ms、確定的、監査可能。

🏗 Architecture

VQL アーキテクチャ

映像解析は一度だけ。VIR にコンパイルした後は、任意のクエリを モデルなしで瞬時に実行できる。知覚誤差は知覚層に閉じ込められ、 問い合わせ層では完全な再現性が確保される。

🔒
知覚誤差の局所化
モデルの不確実性は VIR コンパイル時に集中。クエリ実行層は純粋に 集合演算・時系列演算のみで動作するため、再現性100%が保証される。
⚡
コンパイル一回、クエリは無限
同一映像に対して新しいクエリを投げるたびにモデルを動かす必要はない。 VIR を一度生成すれば、異なる視点からの問い合わせが <10ms で返る。
🔍
証拠付き・監査可能な結果
各クエリ結果には track_id・時刻範囲・代表フレーム・証拠クリップが付与。 VLM の "答えが変わる" 問題を VIR 上で追跡・検証できる。
📝
宣言的・型付き構文
VQL は型付き構文要素（ENTERS, STAYS, SEQUENCE, PROXIMITY など）を持つ 宣言的言語。パーサ・型チェッカ・実行エンジンを OSS として公開。

SELECT person
FROM VIR("entrance_cam_2h.mp4")
WHERE ENTERS(person, zone("A区域"),
        time_range(from="14:00", to="15:00"))
  AND DURATION(person, zone("A区域")) < 5min
RETURN track_id, enter_t, exit_t, duration,
       evidence_frames(n=2)

🌐 2 Domains

2 つのデモドメイン

言語仕様・パーサ・実行エンジン・デモを OSS として公開。 固定監視映像とスポーツ定点映像の 2 ドメインで動作を検証。

📷
固定監視映像
エントランス・施設内の固定 IP カメラ映像。人物の区域入退室・滞留・動線を長時間クエリ。
VQL クエリ例
SELECT person FROM VIR("entrance_cam")
WHERE ENTERS(person, zone("A区域"), after=time("14:00"))
  AND DURATION(person, zone("A区域")) < 5min
SELECT person FROM VIR("entrance_cam")
WHERE SEQUENCE(ENTERS(person, "受付"),
                ENTERS(person, "A区域"))
  AND DURATION(person, "受付") < 3min
SELECT person FROM VIR("entrance_cam")
WHERE STAYS(person, zone("B区域")) > 30min
  AND TIME_OF_DAY() IN range("13:00","18:00")

スポーツ定点映像
バスケットボール・サッカー等の定点カメラ映像。選手の動線・ゾーン侵入回数・近接関係をクエリ。
VQL クエリ例
SELECT player FROM VIR("basketball_court")
WHERE ENTERS(player, zone("paint_area"), count >= 3)
  AND STAYS(player, "paint_area") > 3s
SELECT pair(p1, p2) FROM VIR("basketball_court")
WHERE PROXIMITY(p1, p2, dist < 1.5m)
  AND DURATION(pair, "three_point") > 2s
SELECT player FROM VIR("soccer_field")
WHERE SEQUENCE(ENTERS(player, "penalty_box"),
                SPEED(player) > 5.0 m/s)
  WITHIN 2s
